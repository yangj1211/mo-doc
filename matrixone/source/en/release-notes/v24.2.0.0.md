# MatrixOne v24.2.0.0 Release Report

We are thrilled to announce the official release of MatrixOne kernel v24.2.0.0 on November 1, 2024.

## What is MatrixOne?

MatrixOne is an AI-driven, cloud-native, hyper-converged database. It uses a compute-storage separation architecture, fully leverages cloud infrastructure, is MySQL-compatible, supports hybrid workloads, and — combined with vector data types and full-text search — efficiently handles multi-modal query and management for generative AI applications.

## Feature overview

This release significantly expands MatrixOne's AIGC capabilities and delivers major improvements in enterprise-grade high availability and disaster recovery.

Highlights of this iteration include: data access to external storage and unstructured data, full-text search, vector-search performance improvements, snapshot backup and point-in-time recovery (PITR), CDC, and primary-standby cluster DR based on log replication — plus further MySQL compatibility gains. With these features, MatrixOne is steadily becoming an ideal choice for enterprises building AI-driven, intelligent data management platforms.

## Target scenarios

### Generative AI

MatrixOne's hyper-converged database provides strong multi-modal data support, real-time retrieval, and intelligent data processing for generative AI — forming a core infrastructure layer for GenAI workloads. In text / image generation, MatrixOne delivers efficient data management, vector and hybrid retrieval, Python-UDF-based data cleaning and preprocessing, and GPU-accelerated real-time inference.

### Time-series / IoT

In modern IoT, hundreds of millions of devices continuously ingest TB-level data daily. MatrixOne handles IoT workloads with millisecond-level concurrent writes and fast lookups, plus strong scalability to absorb peak load. Real-time analytics generate insights from massive streams, and seamless integration with ML models enables prediction and anomaly detection — a fit for predictive maintenance, energy-efficiency optimization, and smart monitoring.

### Hybrid workloads

In OA / ERP / CRM systems, traditional single-node databases struggle under peak load — especially at monthly / quarterly closes requiring frequent analytics and real-time reports. MatrixOne's hybrid-workload support means one database serves both OLTP and analytics, keeping response fast under high concurrency. Scalability lets workloads grow seamlessly.

### Enterprise SaaS

Enterprise SaaS must balance multi-tenancy needs. MatrixOne natively supports multi-tenancy with workload isolation, independent scaling, and unified management — lowering operational cost, ensuring isolation, and improving ops efficiency.

## Key new features

### Multi-modal data management

MatrixOne adds direct access to external object storage, remote filesystems, and local filesystems via the `Stage` object, plus direct access to files on storage systems via the `datalink` type. This is particularly useful when building data pipelines for GenAI applications with MatrixOne — it significantly lifts developer productivity and lowers operational costs.

### Full-text index for TEXT / JSON data

Creating full-text indexes on JSON or TEXT columns boosts performance for AIoT-style workloads. Combined with MatrixOne's JSON data type, it reduces data redundancy and raises MatrixOne's competitiveness in AIoT scenarios.

### Vector search

This iteration tunes vector-search performance, enabling fast distance-based retrieval across large vector datasets — critical for LLM- and RAG-based generative AI applications.

### Snapshot-based backup and restore

By creating a data snapshot of a cluster or tenant, you can capture database state at a specific point in time and quickly restore it after failures or emergencies. Snapshots have minimal performance impact, ensure consistency for complete restores, and support cross-tenant restore — strengthening disaster-recovery capability.

### Primary-standby cluster DR based on log replication

Log replication keeps the standby cluster in sync by shipping transaction logs from the primary, delivering high-availability and DR capabilities. When the primary fails, the standby can take over quickly, keeping the business online.

### Point-in-time recovery (PITR)

By recording all changes after an initial snapshot, PITR lets users restore the database to a precise historical moment after a failure, erroneous operation, or data corruption — avoiding loss of critical information. Compared to full backups, PITR cuts backup storage overhead and improves recovery efficiency. It provides flexibility and safety for mission-critical workloads, supports fast recovery, and meets business-continuity and compliance requirements.

### CDC from MatrixOne to MySQL

Capture changes on MatrixOne and replicate them to downstream MySQL in real time — delivering DR from MatrixOne to MySQL. After migrating from MySQL to MatrixOne, you can retain a DR link.

### Table-level publish/subscribe

Previously we supported database-level publish/subscribe; this iteration adds finer-grained table-level pub/sub. On data change, table-level pub/sub syncs specific table updates in real time to subscribers without exposing other tables — offering more flexibility and control than the database-level version.

## Other new features

### SQL statements

- `rename table`
- `create pitr` / `drop pitr` / `alter pitr` / `restore pitr` / `show pitrs`
- Optimized `show publications`
- Optimized `show subscriptions`
- `load data infile` now supports loading in a user-specified column order

### Data types

- Added the `datalink` data type

### Indexes and constraints

- Added full-text index

### Functions and operators

- Added `json_row`, `jq`, `try_jq`, `json_extract_string`, `json_extract_float64` for JSON
- Support arithmetic on dates returned by `now()`

### Tools

- `mo-backup`: PITR management
- `mo_cdc`: CDC task management

### MySQL compatibility

- Case-insensitive string matching in `WHERE` clauses
- Added `Encode()` / `Decode()` functions

## Quickstart

Community users and enterprise developers can deploy MatrixOne with a single command:

```bash
docker pull matrixorigin/matrixone:2.0.0
```

This documentation site also provides architecture guides, installation steps, and development tutorials to help you explore MatrixOne's capabilities. You're welcome to ask questions, discuss, and file feedback at <https://github.com/matrixorigin/matrixone> or in our community chat.

## Known Issues

- In primary-standby DR, the standby does not yet sync external-table or stage data.
- In primary-standby DR, the standby is currently cold-backup only — not readable.
- CDC only supports table-level sync.
- Snapshot backups currently support cluster- and tenant-level backups; restores can be cluster / tenant / database / table level.
- Snapshot and PITR backups cannot recover tenants that have been deleted.
