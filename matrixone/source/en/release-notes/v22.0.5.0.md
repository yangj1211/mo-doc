# MatrixOne v22.0.5.0 Release Report

Congratulations! MatrixOne v22.0.5.0 was officially released on July 18, 2022. In this version, MatrixOne has an independent columnar storage engine capable of supporting HTAP workloads. Cheers!

## Docker

You can pull MatrixOne v22.0.5.0 via docker:

```
docker pull matrixorigin/matrixone:0.5.0
```

## Latest features

- ACID transactions with snapshot isolation.
- `UPDATE`, `DELETE`, and `INSERT INTO ... SELECT ...` syntax.
- `BOOL` and `TIMESTAMP` data types.
- `LEFT / RIGHT / OUTER / NATURAL JOIN`.
- `HAVING` clause.
- Subqueries.
- Common Table Expressions (CTEs).
- `CASE ... WHEN` expressions.
- `Interval` expressions.
- `EXPLAIN` plan tree.
- New aggregate `Any_value`.
- Many other new features.

## Known issues

- v0.5.0-Hotfix may cause data-format incompatibility for some data.
- Frequent insert/update/delete or create/drop-table operations may trigger a *Too many open files* error. Raise *max open files* to fix.
- Running the 1GB TPCH benchmark on machines with less than 64GB memory may OOM.
- Loading `.csv` files over 100GB may hang the system. #3858
- Long mixed runs of TP + AP workloads can trigger system kernel errors. #3947 #3961
