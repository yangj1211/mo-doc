# MatrixOne v22.0.6.0 Release Report

Congratulations! MatrixOne v22.0.6.0 was officially released on November 29, 2022. In this version, MatrixOne is upgraded into a cloud-native HTAP database with compute/storage separation, read/write separation, hot/cold separation, TP/AP separation, and strong elasticity with a complete feature set. Cheers!

## Docker

```
docker pull matrixorigin/matrixone:0.6.0
```

## Latest features

- Distributed ACID transactions with snapshot isolation.
- `TEXT`, `BLOB`, `TIME`, `JSON` data types.
- Permission control — `CREATE` / `GRANT` / `SET ROLE` / `REVOKE` etc.
- `VIEW` support.
- Java / Python / Go connectors plus MyBatis, Spring JPA, SQLAlchemy ORM integration.
- Import CSV and JSON from local filesystem and S3 cloud storage.
- New MatrixOne backup tool `mo-dump`.
- Composite primary keys, unique keys, and auto-increment constraints.
- Added `system_metrics` database in the catalog for instance-state monitoring.
- Added `system` database in the catalog for user statements and system logs.
- Timezone support.
- TLS-encrypted transport.
- Prepared statements: `PREPARE` / `EXECUTE` / `DEALLOCATE`.
- `EXPLAIN ANALYZE` for detailed plan analysis.
- `UNION`, `UNION ALL`, `INTERSECT`, `MINUS` set operators.
- `CREATE TEMPORARY TABLE`.
- `CREATE EXTERNAL TABLE`.
- `PARTITION BY` support.
- Many new system configuration parameters, functions, and system tables for MySQL compatibility.
- New JSON and many DateTime functions.
- `UUID` function.
- Bitwise operators: `&, |, ^, ~, <<, >>`.
- `COMMENT` in DDL.
- User-defined `SET` variables.

## Known issues

- The v0.6 storage format is not compatible with earlier versions — direct upgrade is not possible.
- OLTP concurrent / point-query performance still trails mainstream databases.
- Composite primary keys, unique keys, and secondary indexes don't accelerate queries. #6028
- `INSERT / UPDATE INTO SELECT` over 100MB in distributed environments will fail. #6780
- Continuous loops of `LOAD` can cause OOM. #6793
- Data races can occur in distributed environments. #6855, #6926
- Background tasks can occasionally trigger read/write conflicts with user transactions, aborting them. #6049
- Writing binary files into `BLOB` can fail. #6302
