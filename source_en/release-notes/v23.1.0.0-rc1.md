# MatrixOne v23.1.0.0-RC1 Release Report

We are thrilled to announce the official release of MatrixOne kernel v23.1.0.0-RC1 on August 24, 2023.

This is an important milestone — MatrixOne has essentially implemented the main functionality of a hyper-converged, heterogeneous, cloud-native database. MatrixOne 1.0 GA and MatrixOne Intelligence are coming soon — stay tuned!

## MatrixOne v23.1.0.0-RC1 feature overview

MatrixOne's design and development have always focused on maximizing resource utilization, delivering high price/performance, simplifying data-processing architecture, and lowering the complexity of data processing. v23.1.0.0-RC1 essentially meets those architectural goals — cloud-native + compute/storage separation, hot/cold separation, and fine-grained read/write separation are all fully implemented. The release provides the main features expected of a SQL database, with strong performance across OLTP / OLAP / time-series workloads and basic stream-computing capabilities.

### SQL capabilities

- **Syntax**: highly aligned with MySQL 8.0
- **DDL**: create / alter / drop for databases, tables, views, partitioned tables
- **DML**: insert / update / delete and data import / export statements
- **Basic queries**: group, distinct, filter, sort, limit, regex, etc.
- **Advanced queries**: views, subqueries, joins, set operations, CTEs, window functions, prepared statements
- **Data types**: integer, float, string, date/time, boolean, enum, binary, JSON
- **Aggregates**: AVG, COUNT, MIN, MAX, SUM, and more
- **System functions & operators**: common string, datetime, math functions and operators
- **Indexes & constraints**: primary, unique, not-null, foreign-key, auto-increment, secondary index
- **Stream computing**: create streaming tables; Kafka source ingestion
- **Multi-tenancy**: create and manage tenants inside the database

### Import & export

- Batch loading via `INSERT`, `LOAD`, `SOURCE`
- Data export via `SELECT INTO`, `modump`
- Direct import from local files and object storage

### Transactions

- ACID (Atomicity, Consistency, Isolation, Durability)
- Begin, commit, rollback
- Pessimistic and optimistic transactions — switchable by user
- Snapshot isolation and Read-Committed levels

### Deployment architecture

- Standalone and distributed
- Source, binary, docker, and Kubernetes installation forms
- Public cloud, private cloud, bare metal

### Development & ops

- MySQL-native connectors: Java (JDBC), Python (pymysql), Go (go-sql-driver)
- Common ORMs: MyBatis, Spring JPA, SQLAlchemy, GORM
- MySQL admin tools: Navicat, MySQL Workbench, DBeaver, HeidiSQL
- Standalone and distributed CLI management tool `mo_ctl`
- Built-in database logs and query records, with Grafana integration for real-time cluster monitoring

### Security & access control

- TLS-encrypted transport
- Role-based access control (RBAC)

### Backup & restore

- Logical backup via `modump`
- Physical backup via `mobr`

## Target scenarios

### Lightweight HTAP

OA / ERP / CRM and similar business systems grow past single-node limits as data volumes rise. Many teams add a dedicated analytical database or shard their main DB to handle month-end / quarter-end reporting. With MatrixOne, one database serves both business and analytics, with strong elasticity for seamless growth.

### Real-time analytics

Typical OLAP workloads (dashboards, BI reports) must analyze massive data; at scale they run into performance bottlenecks and slow responses. MatrixOne offers strong analytical performance and scalability — accelerating complex and large SQL queries to near-real-time, improving the agility of decision-making.

### Time-series

Sensors and the internet of things produce massive data — industrial production lines, new-energy vehicles, city surveillance cameras — easily reaching hundreds of TB or PB scale. Traditional databases struggle with this ingest + processing load. MatrixOne handles streaming writes and transforms, with the elasticity to absorb any scale.

### Volatile / innovative workloads

Gaming, e-commerce, entertainment, social, news — high user counts and dramatic workload swings mean peak events need large compute bursts. MatrixOne's fully cloud-native architecture auto-scales with the workload, reducing ops burden.

### Enterprise SaaS

Enterprise SaaS applications typically balance the cost-vs-isolation tradeoff between shared and dedicated database instances. MatrixOne has native multi-tenancy — tenants are workload-isolated and can scale independently, with unified management. A good fit for SaaS.

## Iteration 0.8.0 → 1.0.0

Over two months of iteration, we focused on production-grade feature and usability improvements.

### Key new features

#### Physical backup (Enterprise)

This version introduces physical backup, making it easy to back up and restore the database. Create snapshots and restore to earlier states when needed — preserving integrity and reliability.

#### Stream computing

The final piece of the HSTAP puzzle — the stream-computing framework is now designed and implemented. This iteration adds the ability to create streaming tables, along with a Kafka connector for ingesting streaming time-series data.

#### Recursive CTEs

A recursive CTE repeatedly evaluates an initial CTE to return a subset of data until the full result set is obtained. This iteration's implementation lets users easily handle hierarchical data and build more complex / flexible queries.

### Usability improvements

#### MySQL CREATE TABLE compatibility

MatrixOne now supports MySQL's CREATE TABLE syntax more faithfully — seamless migration from existing MySQL applications without rewriting table definitions.

#### Session migration during scaling

Sessions can be seamlessly migrated when distributed instances scale in or out — no impact on existing sessions or business.

#### Simpler system configuration

Standalone and distributed startup configuration has been significantly simplified — faster to get the database running.

#### mo_ctl tool improvements

The standalone and distributed `mo_ctl` tools are more powerful and easier to use. Distributed `mo_ctl` (Enterprise) now handles deployment, upgrades, and scaling smoothly.

### Other new features

- Added `REPLACE` statement
- `ALTER TABLE` gained more column-attribute changes
- Added `CREATE STAGE` — simplifies import/export flows
- Added `SHOW PROCESSLIST` — see current system status
- Added `enum` type
- Added `Year` type
- Added `TO_DAYS` / `TO_SECONDS` system functions
- Refined `GROUP BY` — supports aliases

### Known issues

- Secondary indexes don't yet accelerate queries
- Memory leaks happen occasionally, potentially causing OOM
- In the distributed version, DN has some single-point-of-failure risk
