# MatrixOne v25.2.1.0 Release Note

We are thrilled to announce the official release of MatrixOne kernel v25.2.1.0 on April 6, 2025.

## What is MatrixOne?

MatrixOne is an AI-driven, cloud-native, hyper-converged database built on a compute-storage separation architecture. It fully leverages cloud infrastructure, is MySQL-compatible, supports hybrid workloads, and — combined with vector data types and full-text search — efficiently handles the multi-modal query and management scenarios of generative AI applications.

![](https://community-shared-data-1308875761.cos.ap-beijing.myqcloud.com/artwork/docs/overview/architecture.png)

## Feature overview

This release brings full support for full-text search and BM25 relevance search, significantly improving the accuracy and efficiency of text-data queries. An optimized `LOAD DATA` command enables efficient bulk ingestion from HDFS, further boosting data-processing performance. Inter-cluster CDC (Change Data Capture) is added to ensure real-time, consistent data synchronization between clusters. HNSW-based vector indexing lands as an experimental feature, providing a new solution for high-dimensional data and vector similarity search. A Python SDK for full-text and vector search is also introduced, lowering the barrier for building intelligent applications.

## Target scenarios

MatrixOne is a fit for the following applications. If your business has these pain points, we welcome you to reach out for a trial.

- **Generative AI** — MatrixOne provides multi-modal data support, real-time retrieval, and intelligent data processing for generative AI, forming a core infrastructure layer for GenAI workloads. In text / image generation scenarios, MatrixOne delivers efficient data management, vector + hybrid retrieval, Python-UDF-based data cleaning and preprocessing, and GPU-accelerated real-time inference — ensuring fast response on large datasets and high-quality output.
- **Time-series / IoT** — In modern IoT, hundreds of millions of devices and sensors continuously ingest TB-level data daily. MatrixOne handles IoT workloads with millisecond-level concurrent writes and fast lookups, plus strong scalability to absorb peak load. Real-time analytics help generate insights from massive IoT streams, and seamless integration with ML models enables prediction and anomaly detection — suitable for industrial predictive maintenance, energy-efficiency optimization, and smart monitoring.
- **Hybrid workloads** — In enterprise OA / ERP / CRM systems, traditional single-node databases struggle to meet peak performance — especially during monthly / quarterly closes requiring frequent analytics and real-time reports. Teams often add a separate analytical database or shard the main one. MatrixOne's hybrid-workload support removes the need for extra systems: one database serves both OLTP and analytics, with real-time analysis keeping response fast under high concurrency. Its scalability lets workloads grow seamlessly.
- **Enterprise SaaS** — Enterprise SaaS development must balance multi-tenancy needs. Teams typically pick between "shared instance" and "per-tenant instance", trading off cost against isolation. MatrixOne natively supports multi-tenancy with workload isolation, independent scaling, and unified management — lowering operational cost, ensuring isolation, and improving ops efficiency.

## New features

### Key new features

- **Full-text search with BM25 relevance**
  Full-text search is now fully supported; BM25 brings accurate relevance ranking and significantly improves text-query efficiency.
- **HDFS data loading**
  An optimized `LOAD DATA` command lets users load large datasets directly from HDFS, meeting big-data ingestion needs.
- **Inter-cluster CDC**
  New inter-cluster CDC enables real-time data sync between MatrixOne clusters, keeping data consistent and available.
- **HNSW-based vector index (beta)**
  HNSW-based vector indexing provides a fresh solution for high-dimensional data and similarity search.
- **Python SDK for hybrid retrieval**
  The MatrixOne Python SDK gives developers a convenient interface for vector, full-text, and hybrid retrieval — cutting complexity for RAG applications.

### Other new features

- Support for `select xx from xx{as of timestamp 'YYYY-MM-DD HH:MM:SS'}`
- Refined PITR syntax
- Refined snapshot syntax

## Known Issues

- CDC tasks currently only support table-level sync
- A PITR covering the sync range must exist before creating a CDC task
- Snapshot and PITR backups cannot recover tenants that have been deleted
- HNSW-type vector indexes do not yet support real-time index updates
