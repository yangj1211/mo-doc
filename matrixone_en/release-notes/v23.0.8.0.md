# MatrixOne v23.0.8.0 Release Report

Congratulations! MatrixOne v23.0.8.0 was officially released on June 30, 2023.

After four months of in-depth development, MatrixOne has made significant strides in performance, scalability, and usability. The highlight of this update is the new Proxy module — it accurately associates different kinds of SQL requests and workload types with the appropriate CN groups, yielding major improvements in tenant isolation, read/write separation, and TP / AP workload separation. v23.0.8.0 is also the last release before MatrixOne goes fully public, and serves as the Beta release of the MatrixOne product. We warmly invite users and developers to try it out — your feedback is greatly appreciated. A big toast to this milestone!

## Docker

- General image:

```
docker pull matrixorigin/matrixone:0.8.0
```

```
docker pull registry.cn-shanghai.aliyuncs.com/matrixorigin/matrixone:0.8.0
```

## Highlights

### Performance

- OLTP: several-fold performance gains on point queries, inserts, and other common OLTP workloads — reaching single-node MySQL levels.
- OLAP: further OLAP performance gains; MatrixOne matches industry-leading OLAP databases on common benchmarks like TPCH.
- Elasticity: seconds-level CN scaling, with near-linear OLTP/OLAP performance growth.

### Usability

- Forward-compatible storage format: from 0.8.0, all future versions preserve compatibility with the v0.8.0 underlying storage format. You don't need to drop old data directories or re-import data on upgrade.
- Simplified deployment and ops: the new `mo_ctl` CLI dramatically simplifies deployment and operations — quickly check the environment, initialize configuration, and install with a single command. Both standalone and distributed deployments no longer require complex Linux or Kubernetes-native commands.
- Better MySQL compatibility: added MySQL-compatible case modes and a SQL-conversion tool from MySQL to MatrixOne. MatrixOne has also added many `information_schema` table structures — significantly lowering the adaptation effort when migrating from MySQL.
- Pessimistic-transaction support: to better match current OLTP application habits, MatrixOne now supports pessimistic transactions, alongside SI and RC isolation levels. You don't need to set up retries for optimistic-transaction conflicts. `SELECT FOR UPDATE` is also supported.

### Latest features

- New window functions: `RANK()`, `ROW_NUMBER()`, `DENSE_RANK()`.
- New `BINARY` type and related functions.
- New publish/subscribe.
- New `PUBLISH` / `SUBSCRIBE` functions.
- Support for `INSERT ... ON DUPLICATE KEY UPDATE`.
- New `Sequence`.
- New `ALTER TABLE`.
- Multi-level foreign keys.
- New `RAND()` function.
- New global-parameter persistence.
- Support for changing passwords to strengthen account security.
- Partial `PARTITION BY` support.

## Known issues

- Data format prior to 0.8.0 is incompatible — direct upgrade is not possible.
- Secondary indexes don't accelerate queries yet.
- Occasional memory leaks may lead to OOM.
- Workload-isolation labels currently only support JDBC.
- TN in the distributed version has some SPOF risk.
- Occasional system hangs under high-concurrency workloads.
- Pessimistic transactions still have some major issues.
