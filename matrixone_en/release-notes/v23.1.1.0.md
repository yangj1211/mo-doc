# MatrixOne v23.1.1.0 Release Report

We are thrilled to announce the official release of MatrixOne kernel v23.1.1.0 on December 29, 2023.

## What is MatrixOne?

MatrixOne is a distributed, hyper-converged, heterogeneous cloud-native database. It aims to provide a cloud-native, high-performance, highly elastic, MySQL-compatible HSTAP database — letting users complete the full data-processing pipeline for transactional, analytical, time-series, and streaming workloads in a single system.

## Highlights

This iteration adds several important capabilities — on top of existing HTAP, it strengthens vector, streaming, and time-series support, further expanding MatrixOne's feature surface and applicability.

### Vector data types

As the data foundation for the 2023 AI / LLM wave, vector types have become standard for general-purpose databases. MatrixOne now supports `vecf32` and `vecf64` types (backed by float32 and float64), along with basic operators and common calculations — sum, L1 / L2 norms, inner product, cosine similarity, etc. Combined with AI model code, users can quickly build an AI application. For example, a RAG-style LLM application (shown below) can embed a knowledge base via OpenAI's embedding API, store vectors in MatrixOne, search for the most similar answers via cosine similarity, and hand the result to an LLM via a prompt for polish.

![](https://community-shared-data-1308875761.cos.ap-beijing.myqcloud.com/artwork/docs/Release-Notes/release-notes-1.1_2.png)

Unlike specialized vector databases, MatrixOne is a general-purpose, highly MySQL-compatible database — the learning curve is minimal. When building AI applications, you can also unify structured data and vector data in a single system.

### Time-series support

To better handle fast-ingestion IoT scenarios, this iteration significantly strengthens time-series capabilities:

- Streaming write capability `LOAD INTO INLINE` — substantially faster than `INSERT INTO`.
- Dedicated time-series tables with a timestamp as primary key, plus arbitrary dimension / metric columns.
- Sliding-window queries on time-series tables — downsample at arbitrary intervals.
- Interpolation for nulls, with multiple strategies.

### Kafka Connector (beta)

On top of the basic stream-computing framework, this iteration adds a Kafka connector. Users can create streaming dynamic tables via `CREATE DYNAMIC TABLE` (append-only) and configure external data sources via `CREATE SOURCE`. Kafka is the first supported source — pointing at a Kafka topic (JSON or protobuf), MatrixOne automatically writes messages into the dynamic table. You can then run window queries on the stream table or join it with other tables.

### User-defined functions (UDFs, beta)

MatrixOne now supports user-defined functions — Python first. In many cases built-in functions don't cover business needs; with UDFs, users can wrap Python logic into SQL functions and call them directly from SQL. Python-based UDFs let you tap into popular libraries like NumPy and scikit-learn, significantly broadening MatrixOne's applicability in data science, ML, and AI.

## Other new features

### DDL

- `INSERT ON DUPLICATE KEY IGNORE`
- `CREATE OR REPLACE VIEW`
- `ALTER SEQUENCE`
- Key / hash partition pruning (beta)
- List / List-column, Range / Range-columns partitioning (beta)

### Indexes and constraints

- Full implementation of secondary indexes — dynamic query acceleration

### Functions

- Python UDFs (beta)
- `SAMPLE` sampling function
- `CONVERT_TZ` timezone conversion
- `SHA2` hash
- `Encode` / `Decode` encoding

### Security and permissions

- Path permissions for `SELECT INTO` via `Stage` objects

### Tools

#### mo_dump (logical backup)

- Dump DDL separately
- Dump multiple databases / tables

#### mo_backup

- Physical backup
- Filesystem and object storage as backup / restore media

#### mo_ctl (standalone)

- Automatic data backup
- Automatic log-table cleanup
- Convert data files from CSV to INSERT / `LOAD DATA INLINE`
- Automatic docker image build
- Standalone deployment via docker

#### mo_ctl (distributed)

- One-command `install` for distributed clusters
- One-command `destroy` for distributed clusters
- Start/stop, upgrade/rollback for MatrixOne clusters
- Install MatrixOne into an existing Kubernetes cluster

#### mo_operator

- Custom S3 certificates
- Backup / restore and backup-data management via API
- Automatically apply optimized Go GC settings
- Enable Python UDFs for MatrixOne clusters
- Prometheus integration for MatrixOne on Kubernetes

### MySQL compatibility

- Significantly reduced the number of MatrixOne-incompatible reserved keywords

## Known Issues

- Secondary indexes do not accelerate `IN` queries
- The Kafka connector only runs in standalone mode
- Occasional system hangs under high-concurrency workloads
- OOM issues under large-data queries
